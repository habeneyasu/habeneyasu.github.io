Architecture & System Design

PFIP — Autonomous Multi-Agent Orchestration (Coordinator-Worker)

flowchart TD U([User via MCP]) --> C[Coordinator] C --> IA[Insights Agent] C --> EA[Expense Agent] C --> SA[Savings Agent] C --> MA[Metrics Agent] IA --> V{4-Layer Validator} EA --> V SA --> V MA --> V V -->|Pass| R([Verified Response]) V -->|Fail| F([SQL Fallback]) C --- DB[(Aurora PostgreSQL)]

Pattern: Coordinator decomposes intent and routes to specialized worker agents. Every LLM output passes a 4-layer validation engine (grounding ±1.5%, coverage, relevance, consistency) before release. Failed validation triggers bounded retry then deterministic SQL fallback — eliminating blind LLM output in financial contexts.

Enterprise RAG Architecture

flowchart LR A[Ingest Docs] --> B[Chunk + Embed] B --> C[(ChromaDB)] C --> D[Semantic Search] D --> E[LLM OpenRouter] E --> F[Response + Citations]
